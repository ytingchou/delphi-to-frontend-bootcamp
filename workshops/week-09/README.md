## Problem

建立企業後台基礎骨架，含 nested layouts、loading/error、基本導航。

## Scope

- `app/(admin)/dashboard/page.tsx`
- `layout.tsx`, `loading.tsx`, `error.tsx`
- 最少 3 條路由

## Constraints

- 遵循 App Router 慣例。
- loading/error 需具備可讀訊息。
- 不可使用 Pages Router。

## Deliverables

- Route tree 截圖
- `routing-decisions.md`
- Cline 對話摘要

## Cline Task

- 要求 Cline 先產生 route tree。
- 請 Cline 給 layout 拆分方案。
- 人工驗證每個檔名慣例。

## Acceptance Criteria

- 路由可正常導航。
- loading/error 可觸發並顯示。
- 結構可維護。
