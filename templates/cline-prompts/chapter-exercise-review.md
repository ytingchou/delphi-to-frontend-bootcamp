# Chapter Exercise Review Prompt (Cline)

## 使用時機

- 你已完成當週 `章節練習（不看答案先做）`。
- 你想要 Cline 幫你做「審查 + 卡關定位」，不是直接重寫答案。

## Prompt 模板（直接貼給 Cline）

```text
你是前端 bootcamp 的 reviewer，請審查我這週的章節練習結果。

背景：
- 週次：Week XX
- 主題：<本週章節主題>
- 我的目標：<本週要達成的能力>

我的實作證據：
1) 程式碼/檔案：
<貼關鍵檔案或 diff>

2) 執行結果：
<貼終端輸出 / 截圖描述>

3) 我遇到的問題：
<貼錯誤訊息與重現步驟>

請用以下固定格式回答：
1. Blocking issues（必修正）
2. Non-blocking improvements（可優化）
3. 最小修正步驟（Step 1/2/3）
4. 我應該如何驗收修正成功

限制：
- 不要整份重寫，只給最小必要修改。
- 先指出風險，再給修正。
- 如果你不確定，請明確標示假設。
```

## 回覆審查清單（人工）

1. Cline 有沒有先指出 blocking 問題。
2. 建議是否可執行（步驟具體、可驗收）。
3. 是否出現過度重構或與需求無關建議。
4. 是否明確區分「確定」與「推測」。
